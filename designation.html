Exploring the Alumni Page

Our client wanted a responsive site with a grid of student thumbnails. By default, the grid would be organized by cohort but the site would have a search function and sorting options to organize by location, discipline, cohort, and availability. In terms of design, we had to use the colors and branding of the existing site. Our objective was to create a user-friendly interface for prospective students and employers and code a search & sort function.

Designing the Site

Given the two-week time constaints, we immediately began sketching and mapping out how each element should be organized. We decided to have a search and filter bar along side a grid with thumbnails, which displayed each student's information when clicked.

Developing the Site

HTML/CSS/JS

My teammate worked on the interface and responsiveness of the site. He coded the animations and microinteractions on the page.

AngularJS/IsotopeJS

I worked with AngularJS to populate the page with each cohort (which contained a list of students) and used Isotope to implement a search and sort feature. One pitfall I came across was using Angular and Isotope together. I regret using an MVC framework like Angular because it over-complicated and interfered with what I was trying to create with Isotope; in retrospect, I would have used simple AJAX calls to draw data from our JSON file. Isotope also sets items in a grid to position: absolute which caused issues with our elements in normal flow. Fortunately, we were able to solve this issue but there are still performance problems that I still want to address.
